# ТЗ: «Бони М» — самоучитель азбуки Морзе (детальная версия)

> Название приложения — **«Бони М»** (по позывному пользователя). Это полное, проверенное техническое задание. Цель — чтобы по нему собралось приложение **без багов, без додумываний и без двусмысленностей**, и чтобы Бонниславу было приятно им пользоваться.
> Таблицы Морзе и тайминги сверены с авторитетными источниками (Википедия / ITU M.1677). Менять их нельзя.

---

## 0. Контекст и принцип
- Единственный пользователь — мужчина 73 лет, г. Пермь, радиолюбитель, позывной **Boney M**. Учит морзянку с нуля. **Имя не захардкожено** — спрашивается на первом запуске и хранится в профиле (см. §2, §7.1, §7.5).
- Не продвинутый пользователь телефона. Интерфейс должен быть крупным, спокойным, прощающим и без скрытых жестов.
- Главный приоритет при любом конфликте решений: **простота и надёжность важнее функций**. Лучше меньше, но безупречно.

## 1. Платформа и технологии (жёсткие требования)
1. **PWA**, устанавливается на Android через «Добавить на главный экран», открывается на весь экран.
2. **Один проект на чистых HTML + CSS + JavaScript. Никаких внешних библиотек, фреймворков, CDN, шрифтов по сети, сборщиков.** Только нативные браузерные API. Это критично для долговечности и офлайна.
3. Звук — **Web Audio API** (генерация тона), без аудиофайлов.
4. Обязательно: `index.html`, `style.css`, `app.js`, `manifest.webmanifest`, `sw.js` (service worker), иконки.
5. **Полностью офлайн** после первого открытия. Service worker кэширует все ресурсы (cache-first) под версионируемым именем кэша (напр. `morse-v1`). При обновлении версии — старый кэш удаляется.
6. Весь прогресс — в `localStorage`, ключ `boni_m_state`, формат JSON. Любое чтение оборачивается в try/catch с откатом к значениям по умолчанию. **Сохранность критична** (см. §13): при первом запуске запрашивать `navigator.storage.persist()` (снижает риск вытеснения данных системой), плюс предусмотреть резервную копию (§7.5). Прогресс обязан переживать перезапуски, обновления приложения и перезагрузки телефона — никогда не начинать «с нуля».
7. Ориентация — **портретная**, интерфейс не должен ломаться при повороте (можно просто оставаться в портретной вёрстке).
8. Хостинг — статика по HTTPS (свой сервер / Netlify / Vercel / GitHub Pages). PWA требует HTTPS.

## 2. Структура состояния (localStorage)
```json
{
  "version": 2,
  "profile": {
    "name": "",                 // спрашивается на старте, редактируется в кабинете
    "callsign": "Boney M",      // предзаполнен, редактируется
    "points": 0                 // накопительные очки опыта (только растут)
  },
  "progress": {                 // прогресс хранится ОТДЕЛЬНО по каждому алфавиту
    "ru": { "learnedCount": 0, "digitsLearned": 0,
            "perChar": { "Е": { "correct": 0, "total": 0 } },
            "recent": [], "parked": [] },  // recent: окно ответов; parked: отложенные знаки (≤2)
    "en": { "learnedCount": 0, "digitsLearned": 0, "perChar": {}, "recent": [], "parked": [] }
  },
  "settings": {
    "alphabet": "ru",           // "ru" | "en"
    "charWpm": 18,              // скорость знаков (Кох)
    "effWpm": 9,                // эффективная скорость (Фарнсуорт), ВСЕГДА <= charWpm
    "keyWpm": 12,               // отдельная, более медленная скорость для режима «Ключ»
    "toneHz": 600,
    "volume": 0.5,
    "showChants": true,         // показывать напевы (только ru)
    "vibration": true
  },
  "streak": { "current": 0, "longest": 0, "lastActiveDate": null }, // даты YYYY-MM-DD, локальные
  "totalSeconds": 0,            // суммарное активное время практики
  "history": [],                // журнал последних сессий (≤30): {date, answers, accuracyPct}
  "milestones": {}              // id вехи -> true
}
```
- `learnedCount` считает **только буквы** (0–32 для ru, 0–26 для en). Цифры — отдельно в `digitsLearned` (0–10).
- `recent` — скользящее окно последних ответов для правила продвижения (§6), ведётся отдельно по алфавиту.
- `points` — «очки опыта»: +1 за верный ответ, +10 за каждую веху. Только растут, ни с кем не сравниваются (см. §8).
- **Миграция:** при загрузке, если ключа нет или `version` старее текущей — мягко привести state к новой схеме, не теряя уже накопленный прогресс. Если `profile.name` пуст — запустить онбординг (§7.1).
```
Если ключа нет или JSON битый — инициализировать значениями по умолчанию выше.

## 4. Звук (Web Audio) — точные параметры
- Один общий `AudioContext`, создаётся/возобновляется **только по первому касанию пользователя** (см. §13, иначе на мобильных звук молчит — это самая частая ошибка).
- Тон: `OscillatorNode` тип `sine`, частота из настроек (по умолчанию **600 Гц**, диапазон 500–800).
- Чтобы не было щелчков: каждый сигнал через `GainNode` с плавными фронтами — атака и спад по **5 мс** (`linearRampToValueAtTime`), полка на громкости из настроек (0–1, по умолчанию 0.5).
- Точка и тире — это включение тона на нужную длительность (см. §5), паузы — тишина.

## 5. Тайминг (СВЕРЕНО, стандарт PARIS + Фарнсуорт)
Базовые соотношения: тире = 3 точки; пауза между элементами знака = 1 точка; между знаками = 3 точки; между словами = 7 точек.

Две скорости:
- **charWpm** (C) — скорость самих знаков (по умолчанию 18).
- **effWpm** (E) — эффективная скорость с растянутыми паузами (по умолчанию 9). Всегда E ≤ C.

Формулы (реализовать ровно так):
```
dit = 1.2 / C                       // длительность точки, секунды (на скорости знаков)
dah = 3 * dit
intraGap = dit                      // пауза между элементами внутри знака

// Растяжка пауз по Фарнсуорту:
ta = (60*C - 37.2*E) / (C*E)        // суммарное «доп.время» на стандартное слово, сек
fUnit = ta / 19                     // единица паузы Фарнсуорта
charGap = 3 * fUnit                 // пауза между знаками
wordGap = 7 * fUnit                 // пауза между словами

// если E >= C (Фарнсуорт не нужен): charGap = 3*dit, wordGap = 7*dit
```
Настройки пользователю показывать словами: ползунок скорости — «медленнее ↔ быстрее» (меняет E в диапазоне 5–15), тон — «ниже ↔ выше». charWpm можно оставить фиксированным 18 или дать отдельный «продвинутый» ползунок 15–22.

**Защита:** интерфейс физически не даёт выставить E > C. Если пользователь поднимает E выше C — либо E упирается в потолок (E = C), либо одновременно поднимается C. Проверять при каждом изменении.

## 6. Методика обучения (метод Коха)
- Учим по одному знаку за раз, на полной скорости знаков (C), с растянутыми паузами (E). Это и есть Кох + Фарнсуорт.
- **Стартовый набор — первые 4 знака сразу** (не 2). Причина: в режиме выбора из кнопок набор из 2 знаков — это угадайка 50/50, правило точности обессмысливается. С 4 знаками выбор уже осмысленный. Дальше знаки добавляются по одному.
- **Правило продвижения** (новый знак открывается, когда выполнены ОБА условия):
  1. общая точность по активному набору ≥ **90%** в скользящем окне последних **30 ответов**;
  2. **новейший** знак отвечен ≥ **10 раз** И его личная точность ≥ **85%** (иначе окно из старых освоенных знаков спрячет проблему с новым).
- **Порядок открытия знаков (русский алфавит)** — реализовать ровно в этом порядке (первые 4 открыты сразу):
  `Е, Т, И, М, А, Н, С, О, У, К, Р, В, Д, Л, П, Г, З, Б, Й, Ь, Ы, Я, Ч, Х, Ж, Ц, Ф, Ю, Ш, Щ, Э, Ъ`
- Для латиницы — классический порядок Коха (первые 4 открыты сразу):
  `K, M, R, S, U, A, P, T, L, O, W, I, N, J, E, F, Y, V, G, Q, Z, H, B, C, D, X`
- Цифры — отдельным треком после освоения ≥ 20 букв, открываются по той же логике (90%/30/10) в порядке `0–9`. Цифры идут в `digitsLearned`, не в `learnedCount`.

## 6.1. Навигация по обучению: повтор, возврат, пропуск
Модель важно понимать: **фиксированных пронумерованных «уроков» нет.** Есть один постепенно растущий набор знаков (Кох). «Листать уроки вперёд-назад» не нужно и вредно — это ломает методику и создаёт пробелы. Вместо этого три понятных действия:

1. **Повторить (внутри занятия).** Кнопка «Повторить знак» переигрывает текущий знак сколько нужно. Дополнительно — «ещё медленнее»: временно снижает E **только для текущего знака** (не меняя сохранённых настроек, соблюдая E ≤ C).
2. **Вернуться к пройденному — режим «Повторение».** Отдельный режим: гоняет только уже освоенные знаки, **новые не открывает**. Нужен для удержания в памяти (в 73 года это критично) и спокойной уверенности. Доступен с Главной и кнопкой в «Учиться». Обновляет статистику `perChar`, но никогда не двигает правило §6 и не открывает новый знак.
3. **Пропустить — мягко, «Отложить знак».** Свободно прыгать вперёд по порядку **нельзя**. Но если новейший знак никак не даётся, приложение **само** (после ~20 попыток при точности < 70%) спокойно предлагает: «Отложить этот знак на потом?». Тогда открывается следующий по порядку знак, а отложенный возвращается в ротацию через несколько знаков. Хранить отложенные в `progress.<алф>.parked` (массив), **одновременно не более двух**.

Базовое поведение: **выход из занятия в любой момент ничего не теряет** — вернувшись, Боннислав продолжает ровно с того же активного набора (всё в `localStorage`). Просмотреть любой знак вне обучения всегда можно в Справочнике (§7.4).

## 7. Экраны и поведение
Нижняя навигация — 4 крупные кнопки с подписями и иконками: **Главная · Учиться · Ключ · Справочник**. Активный раздел подсвечен. Везде кнопка «назад» очевидна, без жестов.

### 7.1. Главная
- Приветствие по имени из профиля: при первом входе после онбординга «Здравствуйте, {имя}!», далее «С возвращением, {имя}!». Если имя ещё не задано — сначала онбординг (ниже).
- В шапке — чип с позывным **Boney M** и аватар героя (`02-portret`). **Тап по шапке/аватару открывает личный кабинет (§7.5).**
- Крупная иллюстрация героя `01-zastavka`.
- Текущий ранг и прогресс (см. §8): «Освоено знаков: 7 из 32», полоса.
- «Дни в эфире»: текущая серия.
- Одна большая кнопка **«Продолжить обучение»** → ведёт в «Учиться» к текущему активному набору. Ниже — вторичная кнопка **«Повторение»** (режим §6.1 п.2), активна, когда освоен хотя бы 1 знак.

**Первый запуск (онбординг):**
1. Тёплый экран-приветствие с героем `01-zastavka` и вопросом **«Как вас зовут?»** — крупное поле ввода и кнопка «Дальше». Введённое имя сохраняется в `profile.name`. Поле нельзя оставить пустым; позывной `Boney M` уже предзаполнен (менять не обязательно).
2. Один спокойный экран на 2–3 фразы: «Здесь вы научитесь принимать и передавать морзянку на слух. Начнём с четырёх знаков и будем добавлять новые, когда вы будете готовы.» и кнопка «Начать».
Никакой обязательной настройки скорости/тона — сразу рабочие значения по умолчанию, настройки доступны позже в кабинете.

### 7.2. Учиться (приём на слух — ядро)
Это режим распознавания по методу Коха.
- Приложение проигрывает один знак из активного набора (звук + синхронная визуальная вспышка для слабослышащих).
- Внизу — крупные кнопки-варианты: знаки активного набора. **Всегда минимум 4 кнопки.** Если в наборе > 8 знаков — показывать 6–8 кнопок: обязательно правильный знак + новейший знак + случайные из набора. **Позиция правильного ответа всегда случайна.** Один и тот же знак не должен быть целью более 2 раз подряд.
- Пользователь нажимает кнопку. Во время проигрывания знака ввод заблокирован, чтобы нельзя было ответить «вслепую» и чтобы звуки не накладывались.
  - **Верно:** мягкий звук успеха, кнопка зеленеет, на ~1.2 сек появляется герой `03-радость` с подписью «Верно!», затем **автоматически** проигрывается следующий знак. Обновить perChar и recent.
  - **Неверно:** мягкий нейтральный звук (не резкий), правильная кнопка подсвечивается, показать код и напев этого знака, дать переслушать. Следующий знак — по кнопке «Дальше» (не автоматически, чтобы человек успел разглядеть). Без негативных формулировок.
- Кнопки «Повторить знак», «Ещё медленнее» (см. §6.1 п.1), «Помощь» (код+напев текущего набора) и «Пауза/выход» доступны всегда. Переключатель «Повторение» включает режим из §6.1 п.2 (только освоенные знаки, без открытия новых).
- Если новейший знак буксует (≥ 20 попыток, точность < 70%) — мягко показать предложение «Отложить этот знак на потом?» (§6.1 п.3).
- При достижении правила §6 — спокойная плашка: «Открыт новый знак: Х (`код`, напев)», затем он добавляется в набор.
- **Завершённая сессия** = пользователь дал ≥ 15 ответов за заход. По её завершении обновляются streak (§8) и totalSeconds. `totalSeconds` накапливается как реально проведённое активное время в режимах «Учиться» и «Ключ» (суммировать дельты, паузить при уходе со вкладки).

### 7.3. Ключ (передача / отстукивание)
- Большая кнопка-«ключ» на нижние 40% экрана (зона нажатия ≥ её всей площади). Нажатие = тон звучит, пока палец прижат.
- **Все пороги считаются от `keyDit = 1.2 / keyWpm`** (отдельная, более медленная скорость `keyWpm`, по умолчанию 12, регулируется). Брать пороги от скорости приёма 18 WPM нельзя: там dit ≈ 66 мс и пороги ~130 мс — отстучать это в 73 года невозможно. При keyWpm = 12 dit = 100 мс — комфортно и прощающе.
- Распознавание элемента по длительности удержания:
  - удержание ≤ `2 * keyDit` → точка; больше → тире.
- Распознавание границ по паузе после отпускания:
  - пауза ≥ `3 * keyDit` и < `7 * keyDit` → конец знака (декодировать и показать букву);
  - пауза ≥ `7 * keyDit` → конец слова (показать пробел).
  - пауза < `3 * keyDit` → это пауза между элементами одного знака, продолжаем накапливать.
- На экране в реальном времени: накопленные `.`/`-`, и распознанная буква. Сверху подсказка-цель (например «отстучите: М») с показом эталона по кнопке «послушать образец».
- Тон при нажатии — тот же Web Audio (см. §4), длительность определяется реальным удержанием.
- Тактильная отдача при нажатии (`navigator.vibrate`, если включено и поддерживается; обязательно с проверкой наличия API).
- Режим прощающий: пороги с запасом, никаких штрафов; цель — удовольствие и моторная память.

### 7.4. Справочник
- Переключатель **РУС / ENG** вверху.
- Сетка знаков. Тап по знаку → крупная карточка: буква, `точки-тире` визуально (крупные • и —), напев (для рус, если включено), кнопка «Послушать». Карточка закрывается тапом по фону или крестиком.
- Отдельная вкладка/раздел «Цифры» и «Дополнительно» (пунктуация).
- Ничего не заблокировано — справочник доступен весь всегда (в отличие от обучения).

## 7.5. Личный кабинет «Бортжурнал»
Открывается тапом по шапке/аватару на Главной. Оформлен как личный радиолюбительский журнал — тепло и с достоинством, без детских элементов.
- **Профиль:** аватар героя (`02-portret`), **имя** (редактируемое поле — можно поправить в любой момент) и **позывной** (редактируемое, предзаполнено `Boney M`). Изменения сразу сохраняются.
- **Очки опыта** (`points`): крупно, как счётчик-одометр. Только растут, ни с кем не сравниваются.
- **Сводка:** текущий ранг (§8), освоено знаков (X из 32), цифры, дни в эфире (текущая/лучшая), суммарное время в эфире.
- **Журнал сессий** (`history`): простой список последних занятий — дата, сколько ответов, точность. Спокойный, без оценок и осуждения.
- **Вехи:** список достижений (§8) с отметкой полученных.
- **Настройки** (здесь же, отдельным блоком): скорость («медленнее ↔ быстрее»), тон («ниже ↔ выше»), громкость, напевы (вкл/выкл), вибрация (вкл/выкл), алфавит РУС/ENG. Всё крупными понятными контролами.
- **Резервная копия (сохранность):**
  - «Сохранить копию» — выгружает весь state одним небольшим JSON-файлом (через ссылку/`Blob`-скачивание).
  - «Восстановить из копии» — загружает такой файл и восстанавливает прогресс (с подтверждением перезаписи).
  - Это страховка на случай очистки браузера/смены телефона. Сыну достаточно раз сделать копию и хранить.
- **Сброс прогресса** — внизу, неприметно, с подтверждением («Точно начать заново?»).
## 8. Геймификация (по-взрослому, без детскости)
- **Ранги** по числу освоенных знаков (`learnedCount` из 32):
  - 0 — «Первый сигнал»
  - ≥ 4 — «Юный радист»
  - ≥ 10 — «Радист»
  - ≥ 18 — «Уверенный приём»
  - ≥ 26 — «Опытный оператор»
  - = 32 и средняя точность ≥ 90% — «Мастер ключа»
- **Дни в эфире**: считать календарные дни с ≥ 1 завершённой сессией. **Логика (по локальной дате `YYYY-MM-DD`):** при завершении сессии сравнить сегодня с `lastActiveDate`. Та же дата → ничего не менять. Ровно вчера → `current += 1`. Старше или null → `current = 1`. Всегда обновлять `longest = max(longest, current)` и `lastActiveDate = сегодня`. Пропуск дня просто сбрасывает серию на следующей сессии — **без укоряющих сообщений, без «вы потеряли»**. Показывать «текущая серия» и «лучшая серия».
- **Вехи** (разовые, показывать тёплую плашку с героем `03-радость`):
  - Освоены первые 4 знака
  - 10 минут в эфире суммарно
  - Принят на слух свой позывной (спецдрилл, см. §9)
  - Освоены все цифры
  - Освоена половина алфавита (16)
  - Освоен весь алфавит
- **Очки опыта** (`points`, §2/§7.5): +1 за верный ответ, +10 за веху. Только накапливаются, видны в кабинете как одометр.
- **Можно** только неконкурентные поощрения (ранги, очки-одометр, дни в эфире, вехи). **Нельзя:** «жизни», таймеры на скорость, рейтинги/лидерборды, сравнение с кем-либо, push-уведомления, любые соревновательные механики.

## 9. Персонализация «для него»
- Имя (из `profile.name`, введено на старте) и позывной (`profile.callsign`, по умолчанию `Boney M`) используются в приветствиях, шапке и кабинете. Имя и позывной редактируются в кабинете (§7.5).
- **Спецдрилл «Свой позывной»** (открывается, когда освоены нужные латинские знаки B, O, N, E, Y, M): приложение проигрывает позывной `Boney M` целиком, как радиограмму, пользователь подтверждает/повторяет. Веха при первом верном приёме.
- В латинском режиме среди тренировочных «слов» использовать радиолюбительские: `QTH, QRZ, QSO, 73, CQ` — это сразу читается как «сделано тем, кто понимает хобби».
- Тон всех текстов — тёплый, уважительный, на «вы», без инфантильности и без сюсюканья.

## 11. Доступность (под 73 года — обязательно)
- Базовый шрифт **≥ 20px**, заголовки 28–34px. Уважать системное увеличение шрифта.
- Контраст текста к фону **не ниже 4.5:1**.
- Зоны нажатия — **минимум 64×64px**, с заметными отступами между ними.
- Никаких свайпов, долгих нажатий, скрытых жестов как единственного способа действия.
- Никогда не полагаться только на цвет: успех/ошибка дублируются текстом и иллюстрацией героя.
- Звук всегда дублируется визуально: индикатор **мигает в ритме самих точек и тире** (короткая вспышка на точку, длинная на тире) — на случай возрастного снижения слуха, чтобы знак можно было «прочитать» глазами.
- Уважать `prefers-reduced-motion`: при включённом — убрать анимации, оставить мгновенную смену состояний.

## 12. Дизайн-система (тёплый светлый стиль — намеренно)
Светлая тёплая тема выбрана осознанно: она лучше читается в возрасте и идеально ложится под прозрачные PNG героя (на тёмном фоне у них был бы светлый ореол).
- Фон: тёплый кремовый `#FBF3E6`. Карточки: белый `#FFFFFF` с мягкой тенью.
- Акцент (латунь/радио): `#C8861E`. Точки-тире в интерфейсе — янтарные.
- Текст основной `#2B2620`, вторичный `#6B6157`.
- Успех `#3E8E5A` (зелёный), ошибка — мягкий тёплый `#C0563B` (не агрессивный красный).
- Шрифт — системный (`system-ui`), крупный, без подключения по сети.
- Скругления, мягкие тени, спокойные плавные переходы. Никакого ретро-китча.

## 13. Крайние случаи и защита от багов (чек-лист реализации)
1. **AudioContext только после жеста.** На первом касании любой кнопки — `audioCtx.resume()`. Без этого на Android/iOS звука не будет. Если контекст ещё не готов — не падать, тихо инициализировать.
2. **localStorage:** всё чтение/запись в try/catch. Битый/отсутствующий state → дефолт. Переполнение квоты → не падать, продолжать в памяти.
3. **Service worker:** на новой версии менять имя кэша и чистить старые; не кэшировать `localStorage`; приложение должно открываться офлайн с первого повторного запуска.
4. **Вибрация:** проверять `('vibrate' in navigator)` перед вызовом; отсутствие — не ошибка.
5. **Wake Lock (по желанию):** во время режима «Учиться» можно запросить `navigator.wakeLock`, обязательно в try/catch; отсутствие API — молча игнорировать.
6. **Быстрые/повторные нажатия** в «Учиться» не должны накладывать звуки друг на друга и не должны засчитываться дважды (блокировать ввод до конца проигрывания знака).
7. **Ключ:** дребезг и сверхкороткие касания фильтровать (минимум ~`0.3*dit`), таймеры на отпускание корректно очищать, чтобы знак не «залипал».
8. **Поворот экрана / сворачивание:** при потере фокуса вкладки останавливать текущий звук (`visibilitychange`), таймеры паузить.
9. **Переключение алфавита ru↔en** не должно ломать прогресс: `perChar`, `learnedCount`, `digitsLearned` и `recent` ведутся раздельно по алфавитам в `progress.ru` / `progress.en` (см. §2). Настройки скорости/тона общие.
10. **Сброс прогресса** — кнопка в настройках с подтверждением («Точно начать заново?»).
11. Все тексты — на русском, без «зависших» английских строк интерфейса (латиница только в режиме en и в позывном/Q-кодах).
12. **Сохранность:** на первом запуске вызвать `navigator.storage.persist()` (в try/catch). Прогресс должен переживать перезапуск приложения, обновление версии (миграция §2) и перезагрузку телефона. Резервная копия (§7.5) выгружается и восстанавливается корректно, восстановление — с подтверждением.

## 14. Критерии приёмки (должно выполняться на реальном Android)
- [ ] Ставится на главный экран, открывается на весь экран, работает офлайн (выключить интернет — всё функционирует).
- [ ] Звук появляется с первого нажатия, без щелчков, тон и громкость регулируются.
- [ ] Скорость регулируется; на медленной скорости паузы заметно длиннее (Фарнсуорт работает).
- [ ] Все коды в Справочнике совпадают с таблицами §3 (проверить выборочно 10 знаков, обязательно Ъ, Э, Щ, Ч, Ш, Й).
- [ ] Режим «Учиться» открывает новый знак по правилу 90%/30/10; прогресс сохраняется после перезапуска.
- [ ] Режим «Ключ» корректно различает точку/тире и декодирует знак; нет «залипания» тона.
- [ ] Реакции героя появляются и исчезают, без артефактов прозрачности на кремовом фоне.
- [ ] Серия «дней в эфире» растёт по дням и не выдаёт укоряющих сообщений при пропуске.
- [ ] Спецдрилл «Boney M» доступен после нужных знаков и засчитывает веху.
- [ ] Шрифт крупный, кнопки большие, контраст высокий; всё работает при системном увеличении шрифта.
- [ ] Старт обучения — сразу 4 знака, в режиме выбора всегда ≥ 4 кнопок, правильный ответ в случайной позиции.
- [ ] Режим «Ключ» комфортно отстукивается на скорости по умолчанию (keyWpm 12), точка/тире и границы знаков ловятся без напряжения.
- [ ] Переключение РУС↔ENG сохраняет раздельный прогресс по каждому алфавиту.
- [ ] Режим «Повторение» гоняет только освоенные знаки и не открывает новые.
- [ ] При буксующем знаке появляется мягкое предложение «отложить»; отложенный знак возвращается позже; одновременно отложенных не больше двух.
- [ ] Выход из занятия и возврат сохраняют тот же активный набор.
- [ ] На старте приложение спрашивает имя; имя сохраняется и используется в приветствии.
- [ ] В кабинете можно поправить имя и позывной; изменения сохраняются и не сбрасывают прогресс.
- [ ] Очки опыта растут за верные ответы и вехи; журнал сессий пополняется.
- [ ] **Главное:** закрыть приложение, выгрузить из памяти, перезагрузить телефон — весь прогресс на месте, не «с нуля».
- [ ] Резервная копия сохраняется в файл и успешно восстанавливается.

## 15. Чего НЕ делать
- Не подключать внешние библиотеки, шрифты, аналитику, рекламу, аккаунты, облако.
- Не добавлять настройки и экраны сверх описанных.
- Не использовать резкие/тревожные звуки и цвета на ошибках.
- Не усложнять навигацию и не прятать действия в жесты.
